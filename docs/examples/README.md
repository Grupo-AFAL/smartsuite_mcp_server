# Examples

Practical tutorials and real-world usage patterns for SmartSuite MCP Server.

## Overview

This section provides hands-on examples that demonstrate common workflows and advanced techniques. Each example includes step-by-step instructions, expected outputs, and explanations.

**Who this is for:**
- New users learning the basics
- Developers building complex queries
- Anyone wanting to see real-world usage patterns

---

## Available Examples

### 1. Basic Workflow

**[Basic Workflow Tutorial](basic-workflow.md)**

Step-by-step tutorial covering fundamental operations:
- Exploring your workspace
- Querying data with filters
- Creating and updating records
- Understanding cache behavior
- Performance optimization basics

**Best for:** First-time users, learning the fundamentals

**Time:** 15-20 minutes

**What you'll learn:**
- How to discover solutions and tables
- How to query records efficiently
- How to create and update data
- How caching improves performance
- Basic filtering patterns

---

### 2. Advanced Filtering

**[Advanced Filtering Patterns](advanced-filtering.md)**

Complex filtering techniques and multi-criteria queries:
- Combining multiple conditions (AND/OR)
- Date range filtering
- Linked record queries
- Multiple select/tag filtering
- User assignment filters
- Common filter mistakes and how to avoid them

**Best for:** Users comfortable with basics, need complex queries

**Time:** 20-30 minutes

**What you'll learn:**
- Complex filter composition
- Operator selection by field type
- Performance-optimized filtering
- Common pitfalls and solutions
- Real-world filter examples

## How to Use These Examples

### 1. Prerequisites

Before starting, ensure you have:
- SmartSuite MCP Server installed and configured
- API credentials set up
- Claude Desktop connected to the server
- Access to a SmartSuite workspace

See [Installation Guide](../getting-started/installation.md) if needed.

### 2. Following Along

Each example includes:
- **User queries** - What to say to Claude
- **Expected responses** - What Claude will return
- **Explanations** - Why it works this way
- **Tips** - Best practices and optimizations

**You can:**
- Copy queries exactly as shown
- Adapt them to your own workspace
- Experiment with variations

### 3. Understanding Output

Examples show realistic outputs, including:
- Record data in plain text format
- Count summaries ("X of Y total")
- Cache hit/miss indicators
- Performance metrics

**Note:** Your actual output will vary based on your workspace data.

Each example follows this format:

**1. Objective**
- What you'll accomplish

- Assumed knowledge
